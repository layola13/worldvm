# 36 数学公式生成与编译管线

## 1. 章节目标
补齐“数学公式生成”这一缺失环节，定义从语义到公式、再到可执行几何表示的完整链路。  
本章与第 30 章（LiteMesh/SDF）互补：第 30 章讲“公式怎么用”，本章讲“公式怎么来”。

## 2. 总体流程
公式生成管线建议固定为 6 步：
1. 语义输入（文本/模板/脚本）。
2. 公式草案生成（LLM 或规则模板）。
3. 语法校验（DSL Parser）。
4. 数学校验（域、边界、稳定性）。
5. 编译与缓存（AOT/JIT）。
6. 执行输出（SDF采样 / 位图光栅化 / AABB导出）。

## 3. 公式 DSL（建议）
定义最小几何 DSL：
1. 基元：`Box`, `Sphere`, `Cylinder`, `Capsule`, `Plane`
2. 组合：`Union`, `Intersect`, `Subtract`
3. 变换：`Translate`, `Rotate`, `Scale`
4. 属性：`material`, `logic_mask`, `tags`

示例：
```text
Shape chair_seat = Box(center=[0,8,0], extents=[6,1,6])
Shape leg_1 = Cylinder(center=[-4,4,-4], r=1, h=8)
Shape chair = Union(chair_seat, leg_1, leg_2, leg_3, leg_4)
```

## 4. 公式生成来源
### 4.1 规则模板生成（优先）
1. 高频对象使用人工模板，稳定性最高。
2. 适合基础资产库初始化。

### 4.2 外挂 LLM 生成（扩展）
1. 通过受限 schema 输出 DSL。
2. 禁止直接输出可执行内核指令。
3. 生成结果必须经过本地验证器。

## 5. 语法校验
Parser 必须校验：
1. 函数名合法性。
2. 参数完整性与类型。
3. 括号与组合树完整性。
4. 禁止未注册操作符。

失败即拒绝进入下一阶段。

## 6. 数学校验
必须执行以下检查：
1. 参数域合法（半径 > 0，尺寸 > 0）。
2. 变换矩阵可逆性/有效性。
3. 组合后包围盒可计算。
4. 数值稳定范围（避免极端溢出）。

## 7. 编译策略
### 7.1 AOT（首选）
1. 将高频公式在构建期编译为内部表示（IR/LUT）。
2. 启动即用，运行时开销低。

### 7.2 JIT（可选）
1. 对动态新增公式做运行时编译。
2. 编译后写入缓存并附校验签名。

## 8. 执行后端
公式编译后可输出三种执行目标：
1. `SDF Evaluate`：点查询与碰撞采样。
2. `AABB Export`：粗检测与分页索引。
3. `Voxel Rasterize`：16³ 临时或持久位图。

## 9. 双轨一致性
为避免“公式与位图”不一致，必须有一致性测试：
1. 同对象在采样点集上比对 SDF 与位图占位结果。
2. 误差超阈值则标记资产异常。
3. 异常资产禁止进入生产字典。

## 10. 缓存设计
缓存键建议：
1. 公式 AST 哈希。
2. 版本号。
3. 精度级别。
4. 目标后端（SDF/AABB/Voxel）。

缓存值：
1. 编译 IR。
2. 预计算 AABB。
3. 可选 16³ 位图快照。

## 11. 与分页系统耦合
1. 公式对象默认不占大量内存，只在激活页按需采样。
2. 高热点对象可下沉为位图缓存，减少重复采样成本。
3. 跨页迁移时只移动实例状态，不复制公式定义本体。

## 12. 与外挂数据工厂耦合
1. 外挂 LLM 可生成“公式草案”而非最终资产。
2. Zig 验证器是最终裁决器。
3. 验证通过后公式才可入 ROM 或缓存层。

## 13. 验收标准
1. DSL 解析成功率（合法输入）接近 100%。
2. 非法输入拦截率 100%。
3. 高频公式编译命中缓存率达目标。
4. 公式与位图一致性误差在阈值内。

## 14. 常见失败模式
1. LLM 输出超 schema。  
处理：自动修复一次，失败则拒收。
2. 公式合法但几何不可用（退化体）。  
处理：数学校验阶段剔除。
3. 动态公式过多导致 JIT 抖动。  
处理：限制每 Tick 编译预算，异步编译。

## 15. 结论
数学公式生成不是“可选优化”，而是 1024³ 虚拟分页下的核心压缩手段。  
通过“DSL + 验证器 + 编译缓存 + 双轨一致性”，可以把公式能力稳定接入 World VM 主链路。
