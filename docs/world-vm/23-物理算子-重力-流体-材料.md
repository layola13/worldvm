# 23 物理算子：重力、流体与材料

## 1. 目标
定义 3D 引擎里的基础物理算子与材料交互规则。  
重点是离散状态推进，不依赖复杂连续方程求解。

## 2. 算子体系
首版物理算子建议：
1. `FORCE_FALL`（重力）
2. `FORCE_PUSH`（推力）
3. `FORCE_DRAG`（阻力/减速）
4. `FLOW_STEP`（流体扩散）
5. `HEAT_STEP`（热状态迁移）
6. `MATERIAL_REACT`（材料反应）

## 3. 重力算子（FORCE_FALL）
### 3.1 触发条件
1. 实例 `mass > 0`
2. 非静态对象
3. 下方无稳定支撑

### 3.2 行为
1. 产生意图 `move(0,-1,0)`。
2. 通过试探相位检测是否可下移。
3. 命中障碍则进入支撑态。

### 3.3 伪代码
```c
if (is_dynamic(i) && has_mass(i) && !has_support(i)) {
    enqueue_intent(i, MOVE_DOWN_ONE);
}
```

## 4. 流体算子（FLOW_STEP）
### 4.1 模型
流体不追求连续介质精度，采用离散扩散规则：
1. 先尝试向下
2. 下方受阻再向侧向扩散
3. 最后考虑回填与稳定

### 4.2 优先序
`DOWN > SIDE > DIAGONAL > HOLD`

### 4.3 性能策略
1. 焦点区细化到体素级。
2. 外围区用体积块近似。
3. 仅对活跃前沿计算，不扫全场。

## 5. 材料模型
材料建议离散枚举：
1. `SOLID`
2. `LIQUID`
3. `GAS`
4. `FRAGILE`
5. `ELASTIC`
6. `COMPOSITE`

材料控制：
1. 碰撞响应方式
2. 热传导行为
3. 破碎阈值
4. 流动阈值

## 6. 材料反应（MATERIAL_REACT）
示例规则：
1. `FRAGILE + high_impact -> BREAK`
2. `SOLID + high_heat -> SOFTEN_STATE`
3. `LIQUID + low_temp -> THICKEN_STATE`

## 7. 热状态迁移（HEAT_STEP）
热状态使用离散档位，不用连续温度场：
1. `COLD`
2. `NORMAL`
3. `HOT`
4. `BURNING`

热状态可触发颜色或硬度变化，用于可解释输出。

## 8. 物理与情绪联动边界
心智寄存器只影响“策略选择”，不篡改基础物理定律。  
例如 Fear 可让系统更保守，但不会让重力失效。

## 9. 计算预算控制
每 Tick 物理算子预算：
1. `max_fall_ops`
2. `max_flow_ops`
3. `max_react_ops`
超限则分批到后续 Tick，避免峰值抖动。

## 10. 回滚与确定性
1. 所有算子执行前记录最小变更快照。
2. 相同输入下必须得到相同输出。
3. 随机分支（若有）必须使用可重放随机种子。

## 11. 验收案例
1. 苹果自由落体并在桌面静止。
2. 杯中水溢出后沿桌面边缘扩散再下落。
3. 高冲击击中脆性材料触发破碎。
4. 热状态变化引起材料属性改变。

## 12. 本章结论
物理算子是 WVM 3D 引擎可信度的基础。  
通过离散规则和预算控制，系统在 CPU 上可稳定运行并保持可解释。
