# 23 物理算子：重力、流体与材料

## 1. 目标
定义 3D 引擎里的基础物理算子与材料交互规则。  
重点是离散状态推进，不依赖复杂连续方程求解。

## 2. 算子体系
首版物理算子建议：
1. `FORCE_FALL`（重力）
2. `FORCE_PUSH`（推力）
3. `FORCE_DRAG`（阻力/减速）
4. `FLOW_STEP`（流体扩散）
5. `HEAT_STEP`（热状态迁移）
6. `MATERIAL_REACT`（材料反应）

## 3. 重力算子（FORCE_FALL）
### 3.1 触发条件
1. 实例 `mass > 0`
2. 非静态对象
3. 下方无稳定支撑

### 3.2 行为
1. 产生意图 `move(0,-1,0)`。
2. 通过试探相位检测是否可下移。
3. 命中障碍则进入支撑态。

### 3.3 伪代码
```c
if (is_dynamic(i) && has_mass(i) && !has_support(i)) {
    enqueue_intent(i, MOVE_DOWN_ONE);
}
```

## 4. 流体算子（FLOW_STEP）
### 4.1 模型
流体不追求连续介质精度，采用离散扩散规则：
1. 先尝试向下
2. 下方受阻再向侧向扩散
3. 最后考虑回填与稳定

### 4.2 优先序
`DOWN > SIDE > DIAGONAL > HOLD`

### 4.3 性能策略
1. 焦点区细化到体素级。
2. 外围区用体积块近似。
3. 仅对活跃前沿计算，不扫全场。

## 5. 材料模型
材料建议离散枚举：
1. `SOLID`
2. `LIQUID`
3. `GAS`
4. `FRAGILE`
5. `ELASTIC`
6. `COMPOSITE`

材料控制：
1. 碰撞响应方式
2. 热传导行为
3. 破碎阈值
4. 流动阈值

## 6. 材料反应（MATERIAL_REACT）
示例规则：
1. `FRAGILE + high_impact -> BREAK`
2. `SOLID + high_heat -> SOFTEN_STATE`
3. `LIQUID + low_temp -> THICKEN_STATE`

## 7. 热状态迁移（HEAT_STEP）
热状态使用离散档位，不用连续温度场：
1. `COLD`
2. `NORMAL`
3. `HOT`
4. `BURNING`

热状态可触发颜色或硬度变化，用于可解释输出。

## 8. 物理与情绪联动边界
心智寄存器只影响“策略选择”，不篡改基础物理定律。  
例如 Fear 可让系统更保守，但不会让重力失效。

## 9. 计算预算控制
每 Tick 物理算子预算：
1. `max_fall_ops`
2. `max_flow_ops`
3. `max_react_ops`
超限则分批到后续 Tick，避免峰值抖动。

## 10. 回滚与确定性
1. 所有算子执行前记录最小变更快照。
2. 相同输入下必须得到相同输出。
3. 随机分支（若有）必须使用可重放随机种子。

## 11. 验收案例
1. 苹果自由落体并在桌面静止。
2. 杯中水溢出后沿桌面边缘扩散再下落。
3. 高冲击击中脆性材料触发破碎。
4. 热状态变化引起材料属性改变。

## 12. 本章结论
物理算子是 WVM 3D 引擎可信度的基础。  
通过离散规则和预算控制，系统在 CPU 上可稳定运行并保持可解释。

## 13. 扩展材质体系（Material / Surface / Medium）
当前 `material_type` 只描述“单体属性”，不足以表达真实接触。  
本章补充三张表：
1. `MaterialType`
2. `SurfaceCondition`
3. `MediumType`

它们共同决定最终的接触响应。

### 13.1 设计原则
1. 不能只保留 `SOLID/LIQUID/GAS` 这类粗分类。
2. 不能把“冰面”“湿地”“抛光地面”错误塞进材质本体。
3. 不能把“布料”“骨骼”“泥地”“深水”混成同一求解路径。
4. 材质决定“本体属性”，表面状态决定“表层响应”，介质决定“后处理通道”。

### 13.2 基础物理族（用于选求解路径）
建议先定义基础物理族：
1. `RIGID_HARD`
2. `RIGID_TOUGH`
3. `RIGID_BRITTLE`
4. `RIGID_ELASTIC`
5. `SOFT_BODY`
6. `CLOTH`
7. `PARTICULATE`
8. `VISCOUS`
9. `LIQUID`
10. `GAS`
11. `SURFACE_ONLY`

说明：
1. `RIGID_HARD`：钢、水泥、石材、冰块等。
2. `RIGID_BRITTLE`：玻璃、陶瓷等。
3. `RIGID_ELASTIC`：橡胶球、轮胎胎面等。
4. `CLOTH`：衣物、旗帜、绳网、软吊带。
5. `PARTICULATE`：沙、松土、雪粒。
6. `VISCOUS`：泥浆、湿泥、胶体。
7. `SURFACE_ONLY`：只参与地表条件调制，不作为独立厚实体求解。

### 13.3 MaterialType（具体材质标签）
建议最小扩展集如下：

#### 金属
1. `IRON`
2. `STEEL`
3. `ALUMINUM`
4. `COPPER`

#### 建筑与石材
1. `CONCRETE`
2. `STONE`
3. `BRICK`
4. `ASPHALT`

#### 脆性材料
1. `GLASS`
2. `CERAMIC`

#### 木材
1. `WOOD_SOFT`
2. `WOOD_HARD`
3. `PLYWOOD`

#### 聚合物与弹性体
1. `RUBBER`
2. `PLASTIC`
3. `FOAM`
4. `SPONGE`

#### 地表与颗粒物
1. `DIRT`
2. `MUD`
3. `SAND`
4. `GRAVEL`
5. `SNOW`
6. `ICE`

#### 液体与流体
1. `WATER`
2. `OIL`
3. `SLURRY`

#### 织物与柔性材料
1. `COTTON_CLOTH`
2. `DENIM`
3. `CANVAS`
4. `LEATHER`
5. `MESH_FABRIC`

#### 人体与生物相关
1. `SKIN`
2. `FLESH`
3. `BONE`
4. `CARTILAGE`

### 13.4 SurfaceCondition（表面状态）
很多差异不是来自材质本体，而是来自表面状态。

建议最小集合：
1. `DRY`
2. `WET`
3. `ICY`
4. `MUDDY`
5. `DUSTY`
6. `ROUGH`
7. `POLISHED`
8. `SOFT_TOP`
9. `CRACKED`

示例：
1. 水泥地：`CONCRETE + DRY`
2. 湿水泥地：`CONCRETE + WET`
3. 结冰水泥地：`CONCRETE + ICY`
4. 覆雪土地：`DIRT + SOFT_TOP`

### 13.5 MediumType（介质类型）
介质不是材质的同义词，它决定额外后处理路径。

建议最小集合：
1. `MEDIUM_AIR`
2. `MEDIUM_HARD_CONTACT`
3. `MEDIUM_SOFT_GROUND`
4. `MEDIUM_WATER_SHALLOW`
5. `MEDIUM_WATER_DEEP`
6. `MEDIUM_MUD_WET`
7. `MEDIUM_SAND_LOOSE`
8. `MEDIUM_SNOW_LOOSE`

说明：
1. `MEDIUM_HARD_CONTACT`：硬碰硬冲量求解。
2. `MEDIUM_SOFT_GROUND`：下陷和滚阻增强。
3. `MEDIUM_WATER_*`：浮力、阻力、飞溅。
4. `MEDIUM_MUD_WET`：粘滞 + 下陷 + 拔出阻力。
5. `MEDIUM_SAND_LOOSE`：快速耗能 + 颗粒位移。

### 13.6 三张表的组合关系
最终接触模型建议为：

`MaterialA x MaterialB x SurfaceCondition x MediumType -> ContactResponse`

含义：
1. `MaterialA`：运动物体本体材质。
2. `MaterialB`：被撞对象材质，或命中的地表材质。
3. `SurfaceCondition`：地表状态修正项。
4. `MediumType`：后处理路径选择。

## 14. 材质对材质响应表（Material Pair）
在扩展材质体系之上，定义统一的 `MaterialPairResponse`。

### 14.1 响应参数
每个材质配对至少输出：
1. `restitution`（回弹系数）
2. `friction_static`（静摩擦）
3. `friction_dynamic`（动摩擦）
4. `damage_factor`（冲击伤害倍率）
5. `sink_factor`（软地形下陷倍率）
6. `splash_factor`（液体飞溅倍率）

### 14.2 最小配对表示例
| Pair | restitution | friction_static | friction_dynamic | damage_factor | sink_factor | splash_factor |
|---|---:|---:|---:|---:|---:|---:|
| IRON x CONCRETE | 0.15 | 0.65 | 0.55 | 0.25 | 0.00 | 0.00 |
| GLASS x CONCRETE | 0.05 | 0.60 | 0.50 | 1.80 | 0.00 | 0.00 |
| BOWLING x CONCRETE | 0.20 | 0.75 | 0.65 | 0.35 | 0.00 | 0.00 |
| IRON x MUD | 0.02 | 0.50 | 0.35 | 0.20 | 0.90 | 0.00 |
| GLASS x MUD | 0.01 | 0.45 | 0.30 | 1.10 | 0.70 | 0.00 |
| BOWLING x MUD | 0.04 | 0.55 | 0.40 | 0.30 | 0.80 | 0.00 |
| IRON x WATER | 0.00 | 0.02 | 0.01 | 0.10 | 0.00 | 0.60 |
| GLASS x WATER | 0.00 | 0.02 | 0.01 | 0.15 | 0.00 | 0.90 |
| BOWLING x WATER | 0.00 | 0.02 | 0.01 | 0.12 | 0.00 | 0.70 |

### 14.3 扩展示例
为了覆盖后续场景，建议继续加入：
1. `BONE x CONCRETE`
2. `SKIN x ASPHALT`
3. `RUBBER x ICE`
4. `COTTON_CLOTH x WATER`
5. `LEATHER x MUD`
6. `STEEL x GRAVEL`

### 14.4 规则优先级
1. 先查配对表（`pair_override`）。
2. 若无配对项，使用通用组合规则（如平均/最小/最大策略）。
3. 所有组合规则必须固定，不允许随平台变化。

## 15. 地表与介质类型系统（Surface / Medium）
“占用/非占用”不足以支撑场景差异，本章补充地表与介质分流。

### 15.1 最小类型集
1. `SURFACE_CONCRETE`（硬地）
2. `SURFACE_SOIL_SOFT`（软土/泥地）
3. `MEDIUM_WATER_SHALLOW`（浅水）
4. `MEDIUM_WATER_DEEP`（深水）
5. `MEDIUM_AIR`（空气）

### 15.2 响应通道
1. 硬地：冲量 + 摩擦 + 回弹。
2. 软地：冲量衰减 + 下陷 + 滚阻增强。
3. 水体：入水冲击 + 浮力 + 速度平方阻力 + 飞溅事件。

### 15.3 与材质配对的关系
接触响应必须由两部分共同决定：
1. `MaterialPair`（物体材质对材质）
2. `Surface/Medium`（命中介质类型）

## 16. 接触响应路径（Contact Response Path）
单次接触建议统一执行以下步骤：
1. `Detect`：获得命中法线、接触点、相对速度。
2. `Classify`：判定命中的是硬地、软地还是液体。
3. `PairResolve`：读取材质配对参数。
4. `ImpulseSolve`：计算法向冲量与摩擦冲量。
5. `DamageEval`：评估损伤与破坏阈值。
6. `MediumPost`：执行下陷、飞溅、浮力等后处理。
7. `Emit`：输出可回放事件与统计数据。

### 16.1 伪代码
```c
contact = detect_contact(a, b_or_env);
surface = classify_surface(contact);
pair = resolve_material_pair(a.material, contact.material);
impulse = solve_impulse(contact, pair, surface);
damage = eval_damage(contact, pair, surface);
apply_medium_post_effect(a, surface, pair, contact);
emit_contact_event(contact, pair, surface, impulse, damage);
```

## 17. 观测接口规范（Headless Assertions）
为支持无头验证，建议每 Tick 统一输出以下观测字段：
1. `max_bounce_height`
2. `impact_speed`
3. `break_event_count`
4. `sink_depth`
5. `splash_intensity`
6. `rest_state`（sleeping/resting/active）
7. `contact_material_pair_id`
8. `surface_medium_type`

### 17.1 事件建议
1. `EVT_CONTACT_BEGIN`
2. `EVT_CONTACT_PERSIST`
3. `EVT_CONTACT_END`
4. `EVT_BREAK`
5. `EVT_SPLASH`
6. `EVT_SINK_UPDATE`

## 18. 本章补充结论
当测试从“是否下落”升级到“不同材质撞不同介质”的阶段时，  
物理算子必须从单体规则升级为“扩展材质体系 + 表面状态 + 介质分流 + 可观测输出”的统一路径。
