# 38 声音宇宙与听觉模拟

## 1. 目标
把听觉从“单次输入字段”升级为独立声音宇宙（Sound Universe），支持可分页、可预演、可具身联动的声音处理。

## 2. 设计边界
1. 处理范围：环境声、语言声、噪声、音乐。
2. 不做高保真音频渲染器；聚焦事件语义与听觉影响路径。
3. 结果输出为门控和生理建议，不替代真实音频引擎。

## 3. 核心数据
1. `SoundEvent`：声源类型、强度、持续时间、方向。
2. `SoundFeature`：节奏、音高区间、动态起伏、语音情绪标签。
3. `AuditoryState`：听觉负荷、熟悉度、刺激阈值、疲劳度。
4. `SoundMemoryAnchor`：与记忆页关联的声音锚点。

## 4. 执行流程（每次激活）
1. 事件注入与特征提取。
2. 声音类别判定（speech/music/noise/environment）。
3. 计算对情绪与生理的增量建议。
4. 写回 Personal Universe 的 `affect_delta/physio_delta`。

## 5. 与影子预演联动
1. 支持在影子沙盒中重放声音片段。
2. 支持“循环播放”短窗预测，评估适应曲线。
3. 预演输出仅作为策略参考，不直接落地行为。

## 6. 内存与调度
1. 建议预算：`16KB ~ 64KB`。
2. 仅在声音事件或内部重放请求时激活。
3. 空闲期保留最小摘要状态，避免常驻开销。

## 7. 最小接口
```c
init_sound_universe(entity_id, cfg)
inject_sound_event(entity_id, sound_event)
step_sound_universe(entity_id, tick_budget)
read_sound_outputs(entity_id, out_delta)
```

## 8. 验收建议
1. 同一声音输入在同一人格配置下输出稳定。
2. 语言声与音乐声可走不同规则分支。
3. 长时重复输入出现可解释的适应衰减。
4. 可导出 trace：事件 -> 规则 -> 输出。

## 9. 本章结论
声音宇宙让听觉具备与视觉、情绪同等的工程地位。  
它是人宇宙可解释具身模拟的必要子系统。
