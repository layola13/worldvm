# 06 外挂 LLM 数据工厂

## 1. 目标
整理 `todo2.md` 的“MiniMax 无限额度 + Zig 物理审计”数据闭环，  
用于构建字典资产与训练样本。

## 2. 角色分工
1. 外挂 LLM（如 MiniMax）：生成语义表达、候选算子、自然语言多样化。
2. Zig 内核：物理真值审计（碰撞、可行性、错误帧）。
3. 数据编排器：将两者拼装成结构化 `jsonl`。

## 3. 三步闭环（推荐）
1. **语义扩张**：同一意图生成多口语版本。
2. **算子草拟**：外挂 LLM 输出候选 `kernel_op`。
3. **真值审计**：Zig 执行并产出 PASS/FAULT 与反馈。

## 4. 样本结构建议
每条样本包含：
1. `instruction`
2. `thought`（可选）
3. `kernel_op`
4. `physics_check`（PASS / 错误码）
5. `feedback`
6. `trace_ref`（可选序列帧索引）

## 5. 质量过滤
1. JSON 结构合法性校验。
2. 算子白名单校验。
3. Zig 审计必须可复现。
4. 错误样本保留原因链，不可只留“失败”标签。

## 6. 正负样本配比
参考 `todo2.md` 思路：
1. 正常路径样本：约 40%。
2. 错误/异常路径样本：约 60%（碰撞、穿透、缺实体、约束冲突）。

## 7. 生成优先级
先刷三类高价值场景：
1. 空间约束（放入、移动、穿透失败）。
2. 交通规则（红绿灯、阻挡、冲突）。
3. 容器与流体（溢出、容量不足、路径受阻）。

## 8. 批处理策略
1. 小批次先验证格式与可执行率。
2. 再扩大并发生成。
3. 每批次都要抽样人工审读。

## 9. 数据落库原则
1. 原始候选与审计结果分库存储。
2. 仅审计通过数据进入“训练候选集”。
3. 冲突与失败样本进入“纠错集”。

## 10. 风险与控制
1. 风险：外挂 LLM 格式抖动。  
控制：严格 schema + 自动修复器 + 拒收策略。
2. 风险：数据看起来合理但物理不成立。  
控制：Zig 必须为最终裁决器。

## 11. 结论
数据工厂的核心不是“多生成”，而是“可验证生成”。  
外挂 LLM 负责丰富表达，Zig 内核负责真理边界。
