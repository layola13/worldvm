# 12 心智引擎与情绪寄存器

## 1. 目标
在不引入黑箱推理的前提下，实现“思考、情绪、决策门控”的统一机制。  
核心方法：影子沙盒 + 四维寄存器 + 策略门控。

## 2. 心智层结构
1. `Shadow Sandbox`：用于预演，不直接改真实状态。
2. `Affect Registers`：`valence/arousal/certainty/control`。
3. `Cognitive Gate`：基于寄存器限制可执行动作集合。
4. `Affect Rules`：寄存器更新规则与触发器。

## 3. 影子沙盒机制
### 3.1 定义
影子沙盒是 Scene32 的轻量副本，用于“先想后做”的快速模拟。

### 3.2 工作流
1. 将候选计划注入影子沙盒。
2. 快速跑有限 Tick。
3. 评估风险、收益、冲突。
4. 通过门控决定是否在真实沙盒执行。

### 3.3 特点
1. 高速：允许低精度近似。
2. 可并行：可同时评估多个候选。
3. 可丢弃：结果只用于决策。

## 4. 四维情绪寄存器
| 寄存器 | 取值 | 含义 |
|---|---|---|
| valence | -128..127 | 正负倾向 |
| arousal | 0..255 | 激活程度 |
| certainty | 0..255 | 判断确定性 |
| control | 0..255 | 可控感 |

## 5. 更新规则示例
1. 威胁接近：`arousal += 40`, `control -= 20`
2. 计划成功：`certainty += 20`, `valence += 10`
3. 连续失败：`certainty -= 30`, `control -= 25`
4. 安全稳定一段时间：`arousal` 按衰减规则下降

## 6. 门控规则示例
1. 当 `arousal > 220`：禁止高复杂度新计划，仅允许避险动作。
2. 当 `certainty < 40`：优先触发 `ASK` 或澄清动作。
3. 当 `control < 30`：禁止激进动作，优先 `FREEZE/RETREAT`。

## 7. 情绪原型
### 7.1 Fear
1. 高 arousal、低 control。
2. 缩小焦点半径，外围降频。
3. 提高 `AVOID/FREEZE` 优先级。

### 7.2 Love/Trust
1. 正 valence、中等 arousal、高 control。
2. 扩展探索范围，提升 `BIND/REPAIR`。
3. 降低防御动作触发阈值。

### 7.3 Sadness
1. 低 valence、低 arousal。
2. 降低整体动作频率。
3. 计划趋向保守与低变化。

## 8. 情绪与计划耦合
计划调度器每 Tick 读取寄存器，决定：
1. 可执行动作白名单。
2. 每 Tick 操作预算。
3. 计划中断阈值。

## 9. 情绪与表达耦合
寄存器还影响输出风格：
1. 高 certainty：直接执行并简洁确认。
2. 低 certainty：先提问澄清再执行。
3. 高 arousal：优先输出安全提示。

## 10. 可解释性要求
每次情绪驱动决策都要记录：
1. 寄存器前后值
2. 触发规则 ID
3. 门控结果
4. 被抑制动作列表

## 11. 伪代码
```c
Decision decide_with_affect(State* st, Plan* p) {
    Affect a = st->affect;
    GateResult g = apply_cognitive_gate(a, p);
    if (!g.allow) return DECISION_BLOCKED;
    return evaluate_in_shadow_sandbox(st, p);
}
```

## 12. 风险与边界
1. 规则过多导致调试困难。  
控制：规则分层、按域启用。
2. 寄存器震荡。  
控制：平滑与滞回。
3. 情绪过拟合场景。  
控制：加入多场景回归测试。

## 13. 指标
1. 风险规避成功率
2. 误拦截率
3. 影子沙盒命中收益
4. 情绪状态稳定度

## 14. 本章结论
心智层不是“神秘模块”，而是可执行规则系统。  
它为 WVM 提供了“先预演、再执行”的安全与效率基础。
