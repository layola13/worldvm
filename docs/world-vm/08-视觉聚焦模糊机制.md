# 08 视觉聚焦模糊机制

## 1. 目标
用“空间降维 + 时间降维”显著降低 CPU 占用，并保持事件关键区的高精度推演。

## 2. 设计原则
1. 焦点内必须精确，焦点外允许近似。
2. 可近似的计算绝不做全精度。
3. 精度提升由事件触发，不做盲目全局刷新。

## 3. 双层空间模型
1. 焦点区（Fovea）：默认 `16x16x16`，位图级碰撞。
2. 外围区（Periphery）：`32³ - Fovea`，AABB/质心近似。

## 4. 双速时间模型
1. 焦点 Tick：`tick_rate = 1`
2. 外围 Tick：`tick_rate = N`（建议 8~16）
3. 静态对象：`tick_rate = 0`

## 5. 焦点选取策略
焦点由以下因素决定：
1. 当前主算子目标（例如被砸对象）。
2. 高威胁实体（危险源）。
3. 任务关键实体（计划 DAG 关键路径）。
4. 视线/指向对象（沟通场景）。

## 6. 唤醒条件（Periphery -> Fovea）
满足任一条件触发升级：
1. 距离阈值触发：`dist < wake_dist`
2. 速度阈值触发：`speed > wake_speed`
3. 威胁阈值触发：`threat_score > wake_threat`
4. 关系阈值触发：关键关系边被波及

## 7. 降级条件（Fovea -> Periphery）
满足全部条件可降级：
1. 无高优先级意图。
2. 连续 K 个 Tick 无冲突。
3. 情绪激活度低于阈值。

## 8. 情绪联动策略
1. Fear 高：缩小焦点半径，外围更低频，形成“隧道视觉”。
2. Love 稳定高：适度扩大焦点，提升关系探索。
3. Sadness 高：整体 Tick 降频，动作计划趋于保守。

## 9. 近似算法建议
### 9.1 外围碰撞
1. AABB 相交检测。
2. 球体半径近似。
3. 阈值内再升级精度。

### 9.2 外围视觉表达
1. 仅存“存在 + 大概位置 + 运动趋势”。
2. 不存细节拓扑。
3. 输出层可用低密度符号显示。

## 10. 预算模型
示意：
1. 全精度 32³：计算量基准 1.0
2. Fovea 16³ + 外围近似：约 0.125 + 少量外围计算
3. 实际下降常可达 60%~90%（依场景）。

## 11. 伪代码
```c
for each instance in scene:
    if in_focus(instance):
        run_full_precision(instance)
    else if should_wake(instance):
        promote_to_focus(instance)
    else if should_sleep(instance):
        run_sparse_update(instance)
```

## 12. 常见错误与修复
1. 误降级：关键对象被降到外围导致漏判。  
修复：关键路径对象强制最小精度级别。
2. 频繁抖动：对象在焦点边界来回升级降级。  
修复：引入滞回阈值（hysteresis）。
3. 外围陈旧：低频更新导致状态过期。  
修复：定期全局轻量校验。

## 13. 指标
1. 焦点对象占比
2. 升级/降级次数
3. 外围漏判率
4. CPU 降幅
5. 用户可感知延迟

## 14. 验收案例
1. “锤子砸玻璃”：焦点保持高精度，外围家具低频。
2. “碎片飞溅打到花瓶”：花瓶从外围被唤醒升级。
3. “长时间静态房间”：系统接近空闲占用。

## 15. 本章结论
视觉聚焦模糊不是 UI 效果，而是 WVM 的核心性能机制。  
它决定了系统能否在 CPU 上长期稳定运行。
