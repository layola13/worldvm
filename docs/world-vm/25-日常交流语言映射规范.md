# 25 日常交流语言映射规范

## 1. 目标
把“日常口语”稳定映射为 WVM 可执行语义。  
本章只解决一件事：`自然语言 -> 结构化 IR` 的标准化映射。

## 2. 映射总原则
1. 先识别“交流意图”，再抽取“对象槽位”，最后生成“动作序列”。
2. 不做自由生成执行；所有执行必须落到已定义模板。
3. 模糊输入必须触发澄清，不允许高风险猜测。

## 3. 日常交流意图分类（v1）
1. `GREET`：问候
2. `ASK_INFO`：询问信息
3. `REQUEST_ACTION`：请求动作
4. `SCHEDULE`：计划安排
5. `CONFIRM`：确认
6. `DENY`：否认/拒绝
7. `THANKS`：感谢
8. `APOLOGY`：道歉
9. `END_CHAT`：结束对话
10. `CLARIFY`：澄清

## 4. 语义槽位定义
每条话语抽取以下槽位：
1. `speaker`（说话方）
2. `listener`（听话方）
3. `intent`（意图）
4. `target`（目标对象）
5. `action`（动作）
6. `time`（时间）
7. `location`（地点）
8. `constraint`（约束：优先级/否定/数量）
9. `confidence`（解析置信度）

## 5. IR 结构（交流域）
```c
typedef struct {
    uint16_t utterance_id;
    uint8_t  intent;
    uint16_t speaker_id;
    uint16_t listener_id;
    uint16_t target_id;
    uint8_t  action_code;
    uint32_t time_slot;
    uint16_t location_id;
    uint8_t  flags;       // question/negation/polite/urgent
    uint8_t  confidence;  // 0..100
} CommIR;
```

## 6. 高频句式映射表（核心）
### 6.1 问候类
1. “你好”“早上好” -> `intent=GREET`, action=`WAVE|OPEN_PALM`
2. “最近怎么样？” -> `intent=ASK_INFO`, target=`status(listener)`

### 6.2 请求类
1. “帮我拿下杯子” -> `REQUEST_ACTION + target=cup + action=GRAB_DELIVER`
2. “请你等一下” -> `REQUEST_ACTION + action=WAIT`
3. “别动” -> `REQUEST_ACTION + action=FREEZE + flags=negation`

### 6.3 安排类
1. “明天下午三点开会” -> `SCHEDULE + action=CREATE_EVENT + time=tomorrow_15:00`
2. “把会议改到周五” -> `SCHEDULE + action=RESCHEDULE + time=Friday`
3. “取消今天晚上的提醒” -> `SCHEDULE + action=CANCEL_EVENT + time=today_evening`

### 6.4 确认/拒绝类
1. “可以”“好的” -> `CONFIRM`
2. “不行”“不要” -> `DENY`
3. “是这个吗？” -> `CLARIFY + target`

### 6.5 礼貌与收尾
1. “谢谢” -> `THANKS`
2. “对不起” -> `APOLOGY`
3. “先这样，回头聊” -> `END_CHAT`

## 7. 映射到执行层（opcode 级）
映射示例：
1. `GREET` -> `FOCUS(listener) + WAVE + ASK(status optional)`
2. `REQUEST_ACTION(拿杯子)` -> `FOCUS(cup) + GRAB(cup) + APPROACH(user) + RELEASE(cup)`
3. `SCHEDULE(创建)` -> `PLAN_CREATE + PLAN_CONFIRM`
4. `DENY` -> `SHAKE_HEAD + DENY`
5. `CONFIRM` -> `NOD + CONFIRM`

## 8. 时间语言规范化
时间短语统一映射到 `time_slot`：
1. “现在/马上” -> `T+0`
2. “待会儿” -> `T+[5,30]min`（低确定性）
3. “明天早上” -> `date+1, 08:00-11:00`
4. “下周一” -> 绝对日期（运行时计算）

若用户时间表达不完整，必须追加澄清问题。

## 9. 否定与约束处理
1. 否定词（不/别/不要）映射 `flags.negation=1`。
2. 数量词（两个、一些）写入 `constraint.quantity`。
3. 强制词（必须、立刻）提升优先级。
4. 委婉词（麻烦、请）不改变语义，只影响输出语气。

## 10. 歧义处理流程
触发歧义条件：
1. target 未识别
2. time 模糊
3. action 与对象不兼容
4. confidence < 阈值

处理顺序：
1. 发起 `CLARIFY`
2. 收集补充槽位
3. 重新编译 IR

## 11. 上下文记忆映射
日常对话大量省略主语/宾语，使用会话上下文补全：
1. “那个给我” -> 从最近焦点对象补 target。
2. “改到明天” -> 从最近计划对象补 action/object。
3. “就这样吧” -> 映射 `CONFIRM + END_CHAT`（按上下文判断）。

## 12. 输出语言模板（反向映射）
执行完成后，输出不要自由发挥，使用模板：
1. 执行成功：`已完成 {action}。`
2. 需要澄清：`你是指 {candidate1} 还是 {candidate2}？`
3. 执行失败：`无法执行 {action}，原因是 {reason}。`
4. 安排确认：`已将 {event} 安排在 {time}。`

## 13. 评测指标
1. 意图识别准确率（Intent Acc）
2. 槽位填充准确率（Slot F1）
3. 歧义触发正确率
4. 计划映射成功率
5. 日常任务完成率

## 14. 最小映射词表建议（首版）
1. 动词：拿、放、给、等、看、说、问、改、删、加、提醒
2. 对象：杯子、门、灯、手机、日程、会议、提醒
3. 时间：现在、今天、明天、下午、晚上、周一到周日
4. 功能词：请、不要、可以、谢谢、对不起

## 15. 示例：完整映射
输入：`明天下午三点提醒我开会。`  
输出 IR：
1. `intent=SCHEDULE`
2. `action=CREATE_REMINDER`
3. `target=meeting`
4. `time=tomorrow_15:00`
5. `listener=self`

执行输出：`已设置提醒：明天 15:00 开会。`

## 16. 本章结论
日常交流不是“文本生成问题”，而是“语义映射问题”。  
只要映射稳定，WVM 就能把沟通直接转成可执行行为与可解释结果。
