# 03 术语与定义

## 1. 术语目的
本章统一关键术语，避免“同词不同义”导致的实现偏差。所有代码、配置、测试命名应尽量使用本章定义。

## 2. 核心术语表
| 术语 | 英文 | 定义 |
|---|---|---|
| 世界虚拟机 | World VM | 用离散状态机驱动物理/认知推演的运行时系统 |
| 实体 | Entity | `16x16x16` 定长基元，包含拓扑与属性 |
| 场景沙盒 | Scene Sandbox | `32x32x32` 局部事件空间，执行推演后销毁 |
| 心跳 | Tick | 离散时间步，系统状态推进基本单位 |
| 关系图 | Relation Graph | 全局权威状态，记录实体关系与语义边 |
| 影子沙盒 | Shadow Sandbox | 思考/预演专用沙盒，不直接改真实世界 |
| 聚焦区 | Fovea | 高精度高频计算区域 |
| 外围区 | Periphery | 低精度低频区域（AABB/质心近似） |
| 变卦 | Mutation | 状态转移结果（拓扑、属性、关系变化） |
| 算子 | Operator | 由 opcode 指定的可执行操作 |

## 3. 内核对象定义
### 3.1 Entity
实体是最小知识与行为载体，属性包括：
1. 局部拓扑位图。
2. 物理属性（质量、硬度、材质等）。
3. 语义属性（颜色索引、角色标签）。
4. 情绪寄存器（可选）。
5. 关系边索引。

### 3.2 Instance
Instance 是实体在某个沙盒中的运行时实例，包含：
1. `entity_id`
2. 当前姿态与位置
3. 临时状态（速度、意图、锁、优先级）
4. 生命周期标记（活跃、休眠、冻结）

### 3.3 Event
事件是系统执行的输入触发，包含：
1. 触发源（用户、外部系统、内部规则）。
2. 目标对象与上下文。
3. 时间戳与优先级。
4. 可选安全级别。

## 4. 时间与一致性术语
1. `Tick`：最小离散时间单位。
2. `Frame`：可视化层面的显示帧，可能包含多个 Tick。
3. `Stable State`：在当前规则下没有可执行意图的状态。
4. `Commit`：将沙盒结果写回全局关系图。
5. `Rollback`：恢复到上一次稳定快照。

## 5. 感知与 LOD 术语
1. `Spatial LOD`：空间精度分层（位图 vs AABB）。
2. `Temporal LOD`：时间精度分层（高频 vs 低频 Tick）。
3. `Wake-up`：外围对象被事件波及后提升计算等级。
4. `Sleep Tick`：外围对象稀疏更新节拍。

## 6. 语言与计划术语
1. `Lexeme`：词法单元。
2. `AST`：语义抽象语法树。
3. `IR`：中间表示。
4. `Plan DAG`：任务计划有向无环图。
5. `Lane`：并行执行通道（左手、右手、语言、视线等）。

## 7. 心智术语
1. `Valence`：情绪正负。
2. `Arousal`：激活水平。
3. `Certainty`：确定程度。
4. `Control`：可控感。
5. `Cognitive Gate`：认知门控，决定哪些计划可执行。

## 8. 安全与审计术语
1. `Policy Guard`：策略守卫，执行前检查。
2. `Trace`：执行链路日志。
3. `Reason Code`：决策原因编码。
4. `Fail-safe`：无法决策时回退到安全默认行为。

## 9. 命名建议
1. 模块前缀统一 `wvm_`。
2. 类型使用名词：`Entity`, `Scene`, `PlanNode`。
3. 动作函数使用动词：`apply_gravity`, `resolve_collision`。
4. 状态量使用可枚举值，避免魔法数字。

## 10. 本章结论
术语统一是可执行规范的前置条件。  
后续章节中的结构体、接口、测试名称均默认采用本章定义。
