# 41 长期记忆的具体实现：记忆页寻址与情感锚点规范

## 1. 记忆页与空间分页的映射 (Spatial-Memory Paging)
长期记忆不再作为独立文件存储，而是挂载在 `27-1024立方虚拟分页架构` 的地址编码下。
- **Memory-Page-ID**: `[X_10bit | Y_10bit | Z_10bit | Event_ID_34bit]`
- **空间相关性加载**：当实体的 `address.zig` 指向某一坐标时，系统计算其所属的 1024 单元格。内核自动预加载该单元格历史上的 `Top-N` 情感强度记忆页。

## 2. 记忆种子 (Memory Seed) 数据结构
在 `src/memory.zig` 中定义的极简存储格式：
```zig
const MemorySeed = struct {
    anchor_tick: u64,           // 情感高峰发生的时间戳
    origin_state_hash: u64,     // 模拟回放前的初始状态指纹
    emotion_spike: [4]f16,      // 记录时刻的情感向量 (Arousal, Pleasure, etc.)
    isa_sequence_ptr: [*]u8,    // 存储 Bagua-ISA 指令流的指针
    isa_length: u32,            // 指令序列长度
    causal_influence: u16,      // 该记忆对未来决策的影响权重
};
```

## 3. 情感锚点驱动的写入流程 (The Spiking Write)
1. **监控**：`mind.zig` 每一 Tick 检测情绪寄存器的导数 $dE/dt$。
2. **触发**：若变化率超过系统定义的 `MEM_TRIGGER_THRESHOLD`，开启临时 Ring-Buffer。
3. **回溯**：将 Ring-Buffer 中前 N 个 Ticks 的 ISA 指令固化为 `MemorySeed`。
4. **分类**：
   - **Trauma (高 Arousal)**：永久锁定，不易衰减，优先级最高。
   - **Routine (低波动)**：进入动态交换区，满 10MB 后优先 LRU 淘汰。

## 4. 记忆回放与“脑内模拟” (Dream-mode Replay)
当 LLM 下发 `memory.replay(seed_id)` 指令：
1. **影子实例创建**：WorldVM 在内存中 Fork 出一个不可见的分身实例（Shadow World）。
2. **确定性重演**：加载 `origin_state_hash` 对应的初始数据，极速灌入 `isa_sequence`。
3. **参数上报**：回放结束后，将 Shadow World 的最终参数（而不是中间过程）反馈给 LLM。
   - *优势*：消耗 0 网络流量，只消耗本地计算力，换取 LLM 对过去经验的深度复盘。

## 5. 10MB 预算内的交换协议 (Memory Swap)
- **L1 (Active)**: 存储在当前 `address.zig` 范围内的 2MB 高频记忆。
- **L2 (Swappable)**: 8MB 的周边空间或高情感强度记忆索引。
- **Swap-Out**: 当 L1+L2 > 10MB，计算 `MemorySeed.causal_influence` 结合时间衰减，将弱记忆持久化到磁盘，释放页内存。

## 6. 与 ISA 指令集的联动
新增指令：
- `MEM_SAVE`：手动强制固化当前窗口的因果种子。
- `MEM_LINK`：将当前记忆与某一特定实体 ID 强关联。
- `MEM_FORGET`：显式擦除某一地址空间的记忆，用于“心智对齐”纠错。

## 7. 评估标准
- **冷启动速度**：从 `address` 切换到记忆页加载完成时间应 < 10ms。
- **因果一致性**：回放产生的 `state_hash` 必须与历史记录完全匹配（误差为 0）。
