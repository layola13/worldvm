# 22 体素空间、变换与碰撞几何

## 1. 目标
给出 3D 引擎几何层的可实现规格：体素布局、坐标映射、变换机制、碰撞流程。

## 2. 体素布局标准
### 2.1 Entity16 位图
1. 固定 `u64[64]`。
2. bit 映射固定（见第 05 章）。
3. 任意实现不得改变线性映射规则。

### 2.2 Scene32 占位图
1. 固定 `u64[512]`。
2. 仅存“当前场景占位”，不存原型细节。
3. 每 Tick 更新时优先写焦点区。

## 3. 坐标系统
1. Scene 局部坐标：`[0..31]`。
2. Entity 局部坐标：`[0..15]`。
3. 实例坐标由 `anchor + local_offset` 计算。
4. 不维护全局永恒坐标，事件结束后只留关系结果。

## 4. 变换模型
### 4.1 平移
通过整格位移实现，避免浮点累积误差。

### 4.2 旋转
采用 LUT 索引：
1. 预烘焙常用朝向（例如 8 向或 24 向）。
2. 实例只切换旋转索引，不重算顶点。

### 4.3 缩放
首版建议不做连续缩放，仅支持离散尺度级别。

## 5. 实例化流程
1. 读取实体原型 ID。
2. 创建实例槽位（位置、朝向、状态）。
3. 将实例投影到 Scene 占位图（焦点优先）。
4. 登记碰撞组与规则组。

## 6. 碰撞分层
### 6.1 粗碰撞（Broad Phase）
1. AABB 或球体包围近似。
2. 快速排除无交集对象。

### 6.2 细碰撞（Narrow Phase）
1. 位图掩码按位与检测。
2. 可选按材料分层检测（硬碰/软碰/流体混合）。

## 7. 碰撞判定输出
碰撞结果至少包含：
1. `pair_id`
2. `penetration_flag`
3. `contact_class`（硬/软/流）
4. `impact_level`
5. `mutate_rule_id`

## 8. 关系型碰撞判定
你在 `chat.md` 里强调“关系优先于距离”：
1. 几何碰撞给出“可能交互”。
2. 关系规则决定“是否允许交互”。
3. 最终结果是几何与关系双重约束。

## 9. 示例：苹果落在桌子上
1. 粗碰撞：苹果 AABB 下移后与桌子 AABB 相交。
2. 细碰撞：苹果位图下移掩码与桌子掩码按位与非零。
3. 触发规则：`FORCE_FALL` 意图取消，状态改为 `REST_ON_TABLE`。
4. 颜色/材料变化按影响等级更新。

## 10. 示例：锤子砸玻璃
1. 场景注入锤子与玻璃。
2. 试探位移命中后计算 `impact_level`。
3. 若超阈值，触发 `BREAK`，玻璃拆分为碎片实例。
4. 碎片进入下一 Tick 继续物理处理。

## 11. 边界处理
1. 越界对象：裁剪或回弹（由规则决定）。
2. 多对象重叠：按优先级逐层解析，防止连锁爆炸。
3. 数值稳定：只用离散步长，不做漂移累积。

## 12. 调试视图
1. AABB 视图
2. 位图切片视图
3. 碰撞对日志
4. 每 Tick 变换轨迹日志

## 13. 本章结论
几何层要“足够简单 + 足够稳定”。  
只要体素布局与碰撞流程固定，3D 执行结果就具备可重复性和可解释性。
