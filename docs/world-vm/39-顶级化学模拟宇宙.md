# 39 顶级化学模拟宇宙

## 1. 目标
建立与物理引擎同级的 Chemical Universe，用统一机制处理嗅觉、味觉、分子扩散、反应、代谢与激素反馈。

## 2. 设计原则
1. 顶级地位：独立状态、规则、步进和观测接口。
2. CPU 友好：小页、离散反应、按需激活。
3. 可解释：每次反应链均可追踪规则与参数。
4. 与物理协同：物理负责宏观运动，化学负责分子级响应。

## 3. 核心模型
1. `ChemicalSignature`：实体的化学标签入口。
2. `MoleculeFeatureVec`：固定维度特征向量（甜/酸/苦/咸/鲜/挥发/毒性等）。
3. `ReactionRuleSet`：离散反应规则表与速率参数。
4. `ChemicalFieldPage`：局部分子场页面（扩散 + 反应状态）。

## 4. 执行管线（每 Tick 或事件触发）
1. `Detect`：接收嗅味觉或摄入事件。
2. `Classify`：区分 orthonasal / retronasal / ingestion。
3. `React`：执行离散反应规则与速率更新。
4. `Integrate`：生成生理、情绪、行为倾向输出。
5. `Commit`：写回人宇宙摘要与 trace。

## 5. 与人宇宙的接口
1. 输入：化学事件、摄入量、接触时长、个体敏感度。
2. 输出：
   1. `affect_delta`
   2. `physio_delta`（如胃部负担、血糖趋势、激素建议）
   3. `behavior_bias`（接近/回避）
3. 支持记忆锚点触发（如特定气味唤起历史事件）。

## 6. 与物理宇宙的协同
1. 物理实体通过 `chemical_signature` 暴露化学入口。
2. 物理“咬合/接触/摄入”事件触发化学反应链。
3. 化学输出可反向影响体力与行为门控。

## 7. 内存与性能建议
1. 建议预算：`8KB ~ 48KB`（按设备档位可调）。
2. 页面建议：`8^3` 或 `16^3` 小页。
3. 仅在化学事件窗口升频，其余时间保持低频摘要。

## 8. 最小接口
```c
init_chemical_universe(entity_id, cfg)
inject_chemical_event(entity_id, chem_event)
step_chemical_universe(entity_id, tick_budget)
read_chemical_outputs(entity_id, out_delta)
```

## 9. 验收建议
1. 同输入同种子下反应链可复现。
2. 嗅觉、味觉、摄入三类路径可区分。
3. 可导出关键观测：反应命中、强度变化、输出建议。
4. 与物理事件联动时无跨页一致性错误。

## 10. 本章结论
顶级化学宇宙把“化学感官与代谢反应”从附加规则提升为正式运行时。  
它让 WVM 从纯物理模拟扩展为物理 + 化学双核驱动的具身系统。
