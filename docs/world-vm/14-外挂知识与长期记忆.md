# 14 外挂知识与长期记忆：因果种子与情感门控协议

## 1. 核心愿景
长期记忆（LTM）不再是静态的文本数据库，而是**动态的因果重建引擎**。利用 WorldVM 的确定性（Determinism），记忆以“因果种子”的形式存储，实现极高的压缩率与物理一致性。

## 2. 记忆三层架构
1. **情节记忆 (Episodic Memory)**：
   - **存储内容**：关键时刻的 `Bagua-ISA` 指令序列（因果种子）+ 初始状态快照。
   - **实现机制**：不记录每一帧的状态，只记录改变世界线的指令。通过 `rewind.zig` 的确定性回放来“重温”经历。
2. **语义记忆 (Semantic Memory)**：
   - **存储内容**：从情节中抽象出的“实体属性拓扑”与“物理常识”。
   - **实现机制**：通过 LLM 对情节进行归纳，写入 10MB 知识库中的动态关系图（MemoryEdge）。
3. **程序记忆 (Procedural Memory)**：
   - **存储内容**：被验证有效的“技能代码块”（ISA 模板）。
   - **实现机制**：将常用的复杂动作序列标记为自定义宏指令，加速后续决策。

## 3. 情感门控机制 (Emotional Gating)
为了符合 100MB 预算，系统拒绝平庸记忆。
- **触发条件**：只有当 `Emotion Registers`（情绪寄存器）的波动率（Spike）超过阈值（如：Fear > 0.7 或 Reward > 0.8）时，才会启动持久化写入。
- **采集窗口**：记录该 Spike 发生前 60 Ticks 和后 30 Ticks 的因果流。

## 4. 记忆分页与寻址 (Memory Paging)
利用 `27-1024立方虚拟分页架构`：
- **空间关联**：记忆与 1024 空间的 `Address Code`（地址编码）强绑定。
- **局部性原理**：当 Agent 物理坐标进入特定页时，该页关联的“记忆种子”自动调入 L2 Cache。
- **预算控制**：活跃记忆页占用不得超过 10MB 知识库配额。

## 5. 记忆衰减与固化算法
```c
// 记忆权重计算模型
float calculate_weight(MemoryEdge* edge) {
    float time_decay = exp(-decay_constant * (current_tick - edge->timestamp));
    float emotional_impact = edge->peak_emotion_amplitude;
    return (time_decay * weight_temporal) + (emotional_impact * weight_emotional);
}
```
- **衰减**：低频、无情感波动的记忆页会被从 L2 交换到持久化存储。
- **固化**：多次被成功调用的记忆会提升 `confidence` 级，进入“常识区”免予衰减。

## 6. 与 LLM 的交互协议
1. **上报**：当感知系统发现当前场景与 LTM 匹配时，向 LLM 返回：
   `{"context": "familiar_scene", "matched_memory_id": 0xAB12, "emotion_echo": "cautious"}`
2. **查询**：LLM 下发 Python 代码：
   `world.memory.recall_by_tag("brittle_objects")` -> 返回因果推断结果。
3. **沉淀**：LLM 周期性下发指令进行记忆整理：
   `world.memory.consolidate(ticks=5000)` -> 将 L1 缓存的情节转化为 L2 语义关系。

## 7. 安全与因果隔离
- 记忆回放必须在“沙盒分身”中进行，不得污染当前 Tick 的世界状态。
- 禁止修改已发生的因果种子（ReadOnly Seeds），确保历史真实性。
