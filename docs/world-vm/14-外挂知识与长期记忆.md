# 14 外挂知识与长期记忆

## 1. 目标
将“推理内核”与“事实记忆”解耦。  
内核保持轻量，事实信息按需检索，不硬塞进 10MB ROM。

## 2. 存储分层
1. `L1 ROM`：10MB 固定知识（规则、模板、高频常识）。
2. `L2 Graph Cache`：本地关系索引缓存。
3. `L3 External Store`：外挂事实库、文档库、事件库。

## 3. 检索原则
1. 先结构化约束检索，再语义扩展。
2. 优先关系一致性，而非纯距离相似度。
3. 返回结果必须可验证、可标注来源。

## 4. 记忆对象模型
```c
typedef struct {
    uint64_t memory_id;
    uint16_t subject_id;
    uint16_t relation_type;
    uint16_t object_id;
    uint8_t  confidence;
    uint64_t timestamp;
    uint32_t source_id;
} MemoryEdge;
```

## 5. 写入策略
1. 只写“已验证结果”。
2. 原始输入与推断结果分开存储。
3. 写入必须携带来源与置信度。
4. 支持过期与衰减。

## 6. 读取策略
1. 查询先命中 L2 缓存。
2. 未命中再查 L3。
3. 查询结果按时间、置信度、来源等级排序。

## 7. 事实冲突处理
1. 同一三元组冲突时不直接覆盖。
2. 维护多版本并记录冲突关系。
3. 由策略层选择“当前生效版本”。

## 8. 记忆衰减
1. 低频、低置信记忆逐步降权。
2. 关键记忆可设置“免衰减”标签。
3. 定期压缩历史，保留摘要索引。

## 9. 与计划系统联动
1. 计划执行前拉取相关记忆约束。
2. 执行后将结果写回记忆层。
3. 失败案例进入“反例记忆”，用于后续规避。

## 10. 与心智层联动
1. 记忆冲突会降低 `certainty`。
2. 高可信支持证据提升 `control`。
3. 风险记忆会提升 `arousal`。

## 11. 外挂检索接口建议
```c
typedef struct {
    uint16_t subject;
    uint16_t relation;
    uint16_t object;
    uint8_t  topk;
    uint8_t  min_conf;
} MemoryQuery;
```

返回：
1. 匹配边列表
2. 来源信息
3. 置信度
4. 冲突标志

## 12. 隐私与安全
1. 记忆数据分级存储。
2. 敏感记忆需访问控制与加密。
3. 提供删除与导出能力。

## 13. 可解释输出
当系统引用外挂记忆时，输出：
1. 用了哪些记忆条目。
2. 来源可信级别。
3. 对结果产生的影响。

## 14. 本章结论
外挂记忆使 WVM 既轻量又可扩展。  
内核负责推理，外挂负责事实容量，二者通过可验证接口协作。
