# 05 实体数据结构规范

## 1. 目标
定义 `Entity16` 的二进制布局、字段语义、版本演进规则。  
本章是 WVM 的“ABI 级规范”，实现语言可以不同，但布局语义必须一致。

## 2. 实体总览
1. 每个实体固定占用 `4096B`（4KB）。
2. 核心区域：
   1. 拓扑区（必需）
   2. 属性区（必需）
   3. 关系区（推荐）
   4. 心智区（可选但建议保留）
   5. 预留区（版本升级）

## 3. 建议布局（v1）
| 区块 | 大小 | 说明 |
|---|---:|---|
| `topology` | 512B | `16*16*16 = 4096 bit` |
| `physics` | 32B | 质量、硬度、材质、摩擦等 |
| `chemistry` | 16B | 化学签名、嗅味觉标签、反应入口 |
| `visual` | 16B | 颜色、反射、透明标记 |
| `semantics` | 64B | 角色标签、语义类别、优先级 |
| `affect` | 16B | valence/arousal/certainty/control |
| `relations` | 256B | 邻接边索引与权重 |
| `behavior` | 128B | 微码入口与规则钩子索引 |
| `reserved` | 3056B | 未来扩展/技能包局部覆盖 |

> 说明：v1 可以不填满预留区，但必须保留字节位，保证结构稳定。
> 校验：以上分区总和固定为 `4096B`。

## 4. 拓扑编码
1. 使用 `u64 topology[64]` 保存 4096 bit。
2. 索引映射采用固定规则：
   1. `bit_index = z + x*16 + y*16*16`
   2. `word = bit_index / 64`
   3. `offset = bit_index % 64`
3. 这样可保证线性扫描与缓存友好。

## 5. 物理字段定义（建议）
```c
typedef struct {
    uint16_t mass;          // 0..65535
    uint16_t hardness;      // 0..65535
    uint8_t  material;      // 枚举：固体/液体/气体/复合
    uint8_t  friction;      // 0..255
    uint8_t  restitution;   // 0..255
    uint8_t  conductivity;  // 0..255
    uint16_t flags;         // bit flags
    uint16_t temp_state;    // 温度离散态
    uint16_t stability;     // 结构稳定度
    uint16_t reserved;
} PhysicsBlock;
```

化学扩展块（建议）：
```c
typedef struct {
    uint32_t chemical_signature; // 分子标签索引或哈希
    uint16_t smell_profile_id;   // 嗅觉配置
    uint16_t taste_profile_id;   // 味觉配置
    uint16_t reaction_rule_set;  // 反应规则组
    uint16_t toxicity_level;     // 0..65535
    uint32_t reserved;
} ChemistryBlock;
```

## 6. 视觉字段定义（10色逻辑调色板）
1. `color_pack`：高 4 bit 主色，低 4 bit 辅色。
2. 建议色表：
   1. 0 黑
   2. 1 白
   3. 2 灰
   4. 3 红
   5. 4 橙
   6. 5 黄
   7. 6 绿
   8. 7 蓝
   9. 8 靛
   10. 9 紫
3. `0xA..0xF` 作为扩展保留值。

## 7. 心智字段定义
```c
typedef struct {
    int8_t  valence;     // -128..127
    uint8_t arousal;     // 0..255
    uint8_t certainty;   // 0..255
    uint8_t control;     // 0..255
    uint8_t mood_flags;  // bit flags
    uint8_t reserved[11];
} AffectBlock;
```

## 8. 关系区定义
关系区建议使用定长槽位：
```c
typedef struct {
    uint16_t target_id;
    uint8_t  relation_type;
    uint8_t  weight;
} EdgeSlot; // 4B
```
`256B` 可容纳 64 条关系边（v1 推荐）。

## 9. 行为钩子定义
行为区不直接存函数指针，存“规则索引”：
1. `on_contact_rule_id`
2. `on_focus_rule_id`
3. `on_threat_rule_id`
4. `on_bind_rule_id`
5. `on_decay_rule_id`
6. `on_sound_event_rule_id`
7. `on_chemical_event_rule_id`

运行时通过规则表查找执行逻辑，避免跨平台 ABI 问题。

## 10. 标志位约定
`flags` 建议位定义：
1. bit0: STATIC
2. bit1: DYNAMIC
3. bit2: BREAKABLE
4. bit3: FLAMMABLE
5. bit4: CONDUCTIVE
6. bit5: TRANSPARENT
7. bit6: INTERACTIVE
8. bit7: RESERVED
9. bit8: HAS_SOUND_PORT
10. bit9: HAS_CHEMICAL_PORT

## 11. 序列化规则
1. 小端序（Little Endian）为标准。
2. 所有段按固定顺序写入。
3. 每实体附带 `entity_crc`，便于校验。
4. 反序列化失败时必须返回错误，不允许静默容错。

## 12. 版本演进规则
1. 不改变既有字段含义。
2. 新字段只追加到预留区。
3. 必须维护迁移脚本。
4. 若发生破坏性变更，提升主版本。

## 13. 调试视图建议
提供三种视图：
1. `topology-ascii`：打印 16³ 切片。
2. `entity-header`：打印关键字段。
3. `edge-dump`：打印关系边与权重。
4. `chem-dump`：打印 chemical signature、profile 与规则组。

## 14. 本章结论
实体结构固定化是 WVM 稳定性的基础。  
只要实体 ABI 稳定，规则、计划、知识库都能持续迭代而不破坏内核。
