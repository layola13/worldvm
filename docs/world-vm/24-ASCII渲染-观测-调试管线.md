# 24 ASCII 渲染、观测与调试管线

## 1. 目标
定义 WVM 3D 引擎的可视化与调试输出方式。  
重点不是“美术渲染”，而是“可观察、可诊断、可回放”。

## 2. 输出层定位
ASCII 渲染在 WVM 中是：
1. 调试工具
2. 状态观测器
3. 解释链可视化终端

它不是图形表现层替代品，但对开发效率至关重要。

## 3. 基础视图类型
1. `Top View`（俯视）
2. `Front View`（正视）
3. `Side View`（侧视）
4. `Slice View`（按层切片）
5. `Focus Map`（焦点/外围热力图）

## 4. 字符映射建议
1. 空体素：`.`
2. 静态固体：`#`
3. 动态固体：`@`
4. 流体：`~`
5. 脆性材料：`*`
6. 焦点对象：`A-Z`

颜色终端可选映射逻辑色盘（10 色）。

## 5. 渲染流程
1. 从 Scene32 读取占位与实例信息。
2. 按视角投影到 2D 网格。
3. 按深度/优先级决定前景字符。
4. 输出帧缓存并刷新终端。

## 6. 与 Tick 同步
1. 默认每 Tick 可输出一帧（调试模式）。
2. 性能模式可每 N Tick 输出一帧。
3. 关键事件（碰撞、破碎、回滚）强制打点输出。

## 7. 调试叠加层
可叠加显示：
1. AABB 边界
2. 焦点半径
3. 速度向量
4. 冲突计数
5. 当前 opcode

## 8. 执行日志联动
每帧可绑定日志片段：
1. `tick_id`
2. `executed_ops`
3. `collisions`
4. `state_delta`
5. `reason_code`

实现“画面 + 指令”同步回放。

## 9. 回放机制
1. 保存关键帧 + Delta。
2. 支持按事件回放。
3. 支持单步、快进、断点。
4. 支持比较两次执行差异。

## 10. 典型调试流程
1. 触发场景（例如锤子砸玻璃）。
2. 观察焦点移动与碰撞时刻。
3. 检查日志中 mutate_rule 是否符合预期。
4. 若异常，回放到前一 Tick 对比状态。

## 11. 性能影响控制
1. 渲染线程与执行线程解耦（可选）。
2. 生产环境默认关闭逐 Tick ASCII 输出。
3. 调试信息按级别开关，避免污染热路径。

## 12. API 建议
```c
void debug_render_scene(const Scene32* s, RenderMode mode);
void debug_dump_tick(const Scene32* s, const TickTrace* tr);
void replay_event_trace(uint32_t event_id);
```

## 13. 验收标准
1. 能清晰显示焦点与外围对象。
2. 能定位碰撞发生 Tick 与对象对。
3. 能对照日志复盘决策路径。
4. 输出开关不影响核心稳定性。

## 14. 本章结论
ASCII 管线是 WVM 的“工程显微镜”。  
它让 3D 引擎和推理行为都变得可见、可验、可调。
